## Practical Overview ##
This practical Assignment is a recreation of an Uber-style homepage built using React. The goal of this assignment was to understand how modern web applications are structured using component-based architecture while also focusing on responsive design and clean UI
Instead of copying everything exactly, I focused on rebuilding the layout in a simple and functional way while maintaining the overall feel of the original Uber interface
## Why i chose this web app? ##
i selected the Uber homepage because it has a clean and modern design that clearly demonstrates how components can be reused and structure and it happens to be aligned with the final project i am creating
## component Structure ##
The application is divided into multiple reusable components:

Navbar – Displays the top navigation menu
Hero – Contains the main heading and booking section
BookingCard – Handles ride input fields and buttons
LocationInput – Reusable input component for pickup and destination
FeatureCard – Displays feature information cards
Footer – Shows footer information

This structure makes the code easier to manage and reuse
## Reusable Components ##
Some components were designed to be reused:
LocationInput
Used for both pickup and destination fields by passing different placeholders.
FeatureCard
Used multiple times to display different features with the same layout
## Responsive Design ##
The layout is responsive and works across different screen sizes:

Desktop → Full layout with proper spacing
Tablet → Adjusted spacing and font sizes
Mobile → Stacked layout with full-width buttons

i used Media queries to ensure the design adapts smoothly across devices
## Implementation Approach ##
I started by breaking the page into smaller components and building a static layout first.
After that, I added styling and then improved the design by making it responsive.
Reusable components like LocationInput helped reduce repetition and made the structure cleaner
